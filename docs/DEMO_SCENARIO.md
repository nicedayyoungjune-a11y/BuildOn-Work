# BuildOn Work Demo Scenario

## 1. 문서 목적

이 문서는 BuildOn Work 화면형 MVP를 고객, 협력사, 개발자, 내부 팀원에게 설명하기 위한 데모 시나리오 문서입니다.

현재 BuildOn Work는 실제 기능 연결 전 단계의 화면형 MVP이며, 근로자·현장소장·운영자가 어떤 흐름으로 사용할 수 있는지 보여주기 위한 시제품입니다.

## 2. 현재 MVP 상태

### 현재 포함된 것

- 랜딩페이지
- 근로자 화면
- 현장소장/건설사 화면
- 관리자/운영자 화면
- 화면 간 이동 구조
- 모바일 가독성 보완
- 고객 관점 문구 보완
- Vercel 배포

### 현재 포함되지 않은 것

- 회원가입
- 로그인
- 권한 구분
- DB 저장
- Supabase 연결
- 실제 일자리 등록
- 실제 지원
- 실제 승인/거절
- 실제 출근 처리
- 실제 문의 접수
- 알림
- 지도
- 결제
- 송금
- 정산

## 3. 데모 시작 전 안내 문구

데모를 시작하기 전에 다음과 같이 설명합니다.

“현재 보시는 화면은 실제 기능이 모두 연결된 완성 서비스가 아니라, 건설 일자리와 현장 인력관리 흐름을 검증하기 위한 화면형 MVP입니다. 오늘은 근로자, 현장소장, 운영자 관점에서 어떤 화면 흐름으로 사용할 수 있는지를 보여드리겠습니다.”

## 4. 데모 전체 흐름 요약

1. 랜딩페이지에서 서비스 개념 확인
2. 근로자 화면에서 일자리 확인
3. 현장소장 화면에서 인력 요청과 출근 현황 확인
4. 관리자 화면에서 전체 운영 현황 확인

## 5. 랜딩페이지 데모 시나리오

경로:

- `/`

설명할 내용:

- BuildOn Work는 건설 일자리와 현장 인력관리를 위한 플랫폼입니다.
- 근로자는 일자리 화면으로 이동할 수 있습니다.
- 현장소장은 현장관리 화면으로 이동할 수 있습니다.
- 신뢰 근거 섹션은 허위 숫자가 아니라 실제 문제와 검증 흐름 중심으로 구성되어 있습니다.
- 현재는 파일럿 흐름 확인 단계입니다.

확인할 포인트:

- 헤드카피
- 근로자/현장소장 선택 카드
- 신뢰 근거 섹션
- 파일럿 CTA
- FAQ

## 6. 근로자 화면 데모 시나리오

경로:

- `/worker/jobs`
- `/worker/jobs/job-001`
- `/worker/applications`
- `/worker/my`

설명할 내용:

- 근로자는 일자리 목록에서 일당, 근무일, 지역, 직종, 지급 조건을 확인합니다.
- 상세 화면에서 출근 전 필요한 정보를 확인합니다.
- 지원한 일자리 화면에서 지원 상태와 출근 확정 여부를 확인할 수 있도록 설계했습니다.
- 내 정보 화면에서는 선호 지역, 선호 직종, 선호 지급 조건을 확인하는 흐름입니다.

주의 설명:

현재는 실제 지원 기능이 연결되어 있지 않으며, 화면 흐름을 보여주는 단계입니다.

## 7. 현장소장/건설사 화면 데모 시나리오

경로:

- `/company/dashboard`
- `/company/sites`
- `/company/jobs`
- `/company/applicants`
- `/company/attendance`

설명할 내용:

- 현장소장은 오늘 필요한 인원과 출근 현황을 먼저 확인합니다.
- 현장 목록에서 현장별 필요한 직종과 진행 중인 일자리를 확인합니다.
- 일자리 관리 화면에서 모집인원, 지원자, 출근 확정자를 확인합니다.
- 지원자 확인 화면에서 지원 상태와 출근 확정 여부를 구분합니다.
- 출근 현황 화면에서 출근 예정자, 출근 완료, 미출근, 근무 완료 상태를 확인합니다.

주의 설명:

현재는 실제 인력 요청, 승인, 출근 처리 기능이 연결되어 있지 않으며, 상태 확인 중심의 화면형 MVP입니다.

## 8. 관리자/운영자 화면 데모 시나리오

경로:

- `/admin/dashboard`
- `/admin/workers`
- `/admin/companies`
- `/admin/sites`
- `/admin/jobs`
- `/admin/applications`
- `/admin/inquiries`

설명할 내용:

- 운영자는 전체 근로자, 건설사, 현장, 일자리, 지원, 문의 현황을 확인합니다.
- 근로자 현황에서는 선호 지역, 직종, 지급 조건을 확인합니다.
- 건설사 현황에서는 등록 현장과 일자리 수를 확인합니다.
- 현장 현황에서는 현장명, 건설사, 지역, 필요한 직종을 확인합니다.
- 일자리 현황에서는 모집인원, 지원자, 출근 확정자를 확인합니다.
- 지원 내역과 문의 내역은 운영자가 확인할 수 있도록 정리되어 있습니다.

주의 설명:

현재는 실제 승인, 수정, 삭제, 답변 기능이 연결되어 있지 않습니다.

## 9. 고객에게 강조할 핵심 메시지

- 전화 중심 인력관리의 불편을 줄이는 방향입니다.
- 근로자는 일자리 조건을 먼저 확인할 수 있습니다.
- 현장소장은 지원자와 출근 현황을 한눈에 확인할 수 있습니다.
- 운영자는 전체 현황을 확인할 수 있습니다.
- 현재는 화면형 MVP이며, 다음 단계에서 실제 기능을 순차적으로 연결할 수 있습니다.

## 10. 데모 중 피해야 할 표현

피해야 할 표현:

- 이미 실제 지원이 가능합니다.
- 바로 인력 요청이 가능합니다.
- 자동 매칭됩니다.
- 자동 송금됩니다.
- 출근을 보장합니다.
- 당일 지급을 보장합니다.
- 완성된 서비스입니다.
- 지금 회원가입하면 됩니다.

대신 사용할 표현:

- 화면 흐름을 보여주는 단계입니다.
- 실제 기능은 이후 단계에서 연결됩니다.
- 현장 피드백을 받아 우선순위를 정할 예정입니다.
- 파일럿 운영을 통해 검증할 수 있습니다.
