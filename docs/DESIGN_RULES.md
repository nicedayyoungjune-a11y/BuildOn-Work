# Design Rules

## Product Tone

BuildOn Work should feel simple, reliable, and practical.

The interface should be easy for construction daily workers, skilled workers, site managers, and company staff to understand quickly.

## UX Principles

- Mobile-first responsive design.
- Clear primary actions on every screen.
- Short labels and direct language.
- Avoid unnecessary animation.
- Avoid visual complexity that slows task completion.
- Make job status, application status, attendance status, and payment option information easy to scan.

## Landing Page Direction

The landing homepage should communicate:

- What BuildOn Work does
- Why workers should sign up
- Why companies should register sites and jobs
- How the service flow works
- How payment options and expected settlement information are shown
- Why users can trust the service

## Webapp Direction

Worker screens should prioritize:

- Job discovery
- Application
- Confirmed attendance status
- My page information

Company and site manager screens should prioritize:

- Site registration
- Job posting
- Applicant review
- Attendance confirmation
- Attendance status management

Admin screens should prioritize:

- Operational visibility
- Basic management workflows
- Clear tables and filters when implemented
